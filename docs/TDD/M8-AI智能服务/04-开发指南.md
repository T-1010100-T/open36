## 2. 开发环境搭建

### 2.1 前置条件

- Python 3.11+
- PostgreSQL 14+（复用现有实例）
- Redis 7+（复用现有实例）
- Anthropic API Key

### 2.2 本地开发步骤

```bash
# 1. 进入项目目录
cd Open436-AIService

# 2. 创建虚拟环境
python -m venv venv
source venv/bin/activate  # Linux/Mac
# venv\Scripts\activate   # Windows

# 3. 安装依赖
pip install -r requirements.txt

# 4. 配置环境变量
cp .env.example .env
# 编辑 .env 填入实际配置

# 5. 初始化数据库
psql -h localhost -p 55432 -U open436 -d postgres -c "CREATE DATABASE ai_db;"
psql -h localhost -p 55432 -U open436 -d ai_db -f migrations/001_init.sql

# 6. 启动服务
uvicorn app.main:app --host 0.0.0.0 --port 8008 --reload

# 7. 启动Celery Worker（另一个终端）
celery -A app.celery_app worker --loglevel=info --concurrency=2
```

### 2.3 环境变量配置

```bash
# .env.example

# 数据库（复用现有PostgreSQL实例，使用独立数据库名）

# LLM

# 内部服务地址（本地开发用 localhost，Docker环境用容器名）

HOJ_API_URL=http://localhost:6688

# HOJ管理员账号（用于HOJ API鉴权）

# 内部API密钥（必须与现有Python服务的 INTERNAL_API_KEY 一致）
# 现有服务默认值：change-me-in-production（见 Open436-ContentService/.env.example）
```

### 2.4 Docker Compose集成

将M8服务添加到现有 `docker-compose.yml`：

```yaml

        condition: service_healthy
```

## 3. 编码规范

### 3.1 与现有项目对齐

M8服务遵循与现有Django服务一致的编码风格：

| 规范 | 说明 |
|------|------|
| **语言** | Python 3.12（与现有服务一致） |
| **命名** | snake_case（变量、函数、文件名） |
| **注释** | 中文docstring和注释 |
| **响应格式** | `{code, message, timestamp, data}`（复用 `apps/core/responses.py` 模式） |
| **错误处理** | 统一异常 → 统一错误响应 |
| **Git提交** | `feat:`, `fix:`, `docs:` 前缀 |

### 3.2 与Django服务的差异

M8使用FastAPI而非Django，主要差异：

| 维度 | 现有Django服务（M2-M5） | M8 FastAPI服务 |
|------|------------------------|----------------|
| **Web框架** | Django 4.2.7 + DRF 3.14.0 | FastAPI 0.115+ |
| **ORM** | Django ORM（`managed=False`） | SQLAlchemy 2.0+ |
| **异步** | 同步（gunicorn多进程） | 原生async |
| **用户信息** | `UserInfoMiddleware` 中间件 | FastAPI `Depends` 依赖注入 |
| **路由** | `urls.py` + DRF Router | `APIRouter` + 装饰器 |
| **序列化** | DRF Serializer | Pydantic Schema |
| **文档** | drf-spectacular（Swagger） | FastAPI自带OpenAPI |
| **进程管理** | gunicorn + uvicorn worker | uvicorn直接运行 |
| **数据库迁移** | SQL脚本（`managed=False`） | SQL脚本（与现有模式一致） |

> ⚠️ M8的数据库表同样使用 `managed = False` 模式（如果使用Django ORM），或直接用SQLAlchemy手动建表。建表SQL放在 `migrations/` 目录下。

### 3.2 FastAPI规范

```python
# 路由定义示例
from fastapi import APIRouter, Depends
from app.core.responses import success_response
from app.schemas.chat import ChatRequest, ChatResponse

router = APIRouter(prefix="/api/ai", tags=["AI对话"])

@router.post("/chat")
async def chat(
    request: ChatRequest,
    user_id: int = Depends(get_current_admin),  # 依赖注入：验证管理员
):
    """同步对话接口"""
    result = await chat_service.process(request, user_id)
    return success_response(data=result)
```

### 3.3 Agent工具规范

```python
# 工具定义示例
from langchain_core.tools import tool
import httpx

@tool
async def create_post(title: str, content: str, section_id: int) -> dict:
    """创建论坛帖子。

    Args:
        title: 帖子标题，5-100字符
        content: 帖子内容，Markdown格式，10-50000字符
        section_id: 板块ID
    """
    async with httpx.AsyncClient() as client:
        response = await client.post(
            f"{settings.CONTENT_SERVICE_URL}/internal/posts/",
            json={"title": title, "content": content, "section_id": section_id},
            headers={"X-Internal-API-Key": settings.INTERNAL_API_KEY},
            timeout=30.0,
        )
        response.raise_for_status()
        return response.json()
```

### 3.4 Pydantic Schema规范

```python
from pydantic import BaseModel, Field

class ChatRequest(BaseModel):
    """对话请求"""
    message: str = Field(..., min_length=1, max_length=5000, description="用户消息")
    conversation_id: str | None = Field(None, description="会话ID，不传则新建")

class ChatResponse(BaseModel):
    """对话响应"""
    conversation_id: str
    message_id: str
    reply: str
    intent: str
    agent_name: str
    tool_calls: list[dict]
    token_usage: dict
```

## 4. 开发流程

### 4.1 分支策略

```
main ← dev ← feature/m8-xxx
```

- `main`：稳定版本
- `dev`：开发分支
- `feature/m8-xxx`：功能分支

### 4.2 开发顺序

1. **基础设施**：项目骨架、配置、数据库连接、Consul注册
2. **数据层**：SQLAlchemy模型、Pydantic Schema
3. **工具层**：封装M3/M5/HOJ的API调用
4. **Agent层**：Router/Forum/Problem Agent + LangGraph图
5. **API层**：FastAPI路由 + 管理员鉴权
6. **异步层**：Celery任务
7. **集成测试**：端到端测试

### 4.3 Consul服务注册

M8服务启动时需要向Consul注册（与现有Python服务保持一致）：

```python
import consul
import socket

def register_consul():
    """注册服务到Consul"""
    c = consul.Consul(host="consul", port=8500)
    service_id = f"{settings.CONSUL_SERVICE_NAME}-{socket.gethostname()}"
    c.agent.service.register(
        name=settings.CONSUL_SERVICE_NAME,
        service_id=service_id,
        address=socket.gethostname(),
        port=int(settings.SERVICE_PORT),
        check=consul.Check.http(
            url=f"http://{socket.gethostname()}:{settings.SERVICE_PORT}/health",
            interval="10s",
            timeout="5s",
        ),
    )
    return service_id
```

> 参考现有实现：`Open436-SectionService/apps/sections/management/commands/register_consul.py`
