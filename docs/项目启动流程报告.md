# Open436 项目启动流程报告

## 一、项目架构概览

Open436 教育平台采用**混合部署**架构：
- **Docker**：基础设施 + HOJ 判题系统
- **本地运行**：业务服务 + 前端

### 服务清单

| 分类 | 服务名 | 端口 | 运行方式 | 说明 |
|------|--------|------|----------|------|
| **基础设施** | postgres | 55432 | Docker | 主数据库 |
| | redis | 6380 | Docker | 缓存 |
| | consul | 8500 | Docker | 服务注册发现 |
| | kong + kong-db | 8000/8001 | Docker | API 网关 |
| | minio | 9000/9001 | Docker | 对象存储 |
| | etcd | 2379 | Docker | Milvus 元数据存储 |
| | milvus-standalone | 19530 | Docker | 向量数据库（AI 服务依赖） |
| **HOJ 系统** | hoj-mysql | 3308 | Docker | HOJ 专用数据库 |

| | hoj-backend | 6688 | Docker | HOJ 后端 API |
| | hoj-judge | 8088 | Docker | 判题服务 |
| | hoj-vue | 8066 | Docker | HOJ 前端 |
| **业务服务** | auth-service | 8081 | 本地 mvn | 认证服务 (Java) |
| | enrollment-service | 8084 | 本地 mvn | 报名服务 (Java) |

## 二、启动流程

### 步骤 1：启动 Docker 基础设施

```bash
docker compose up -d postgres redis consul kong-db kong minio etcd milvus-standalone
```

等待所有服务健康：
```bash
docker compose ps postgres consul kong milvus-standalone
```

预期状态：
- postgres: `healthy`
- consul: `healthy`（leader elected）
- kong: `healthy`
- milvus-standalone: `healthy`（首次启动需等待 ~90s）

### 步骤 2：启动 HOJ 系统

```bash
docker compose --profile hoj up -d
```

> **首次启动注意**：hoj-judge 容器使用 volume 挂载的 `hoj-judge-app.jar`（已做二进制 patch），首次启动会从数据库读取测试用例并写入共享 volume `hoj-testcase`，供 go-judge 读取。

验证：
```bash
# HOJ 后端 API
curl http://localhost:6688/api/get-problem-list
# 预期返回: {"status":200,"data":{...},"msg":"success"}

# HOJ 前端
curl -o /dev/null -w "%{http_code}" http://localhost:8066/algo/
# 预期返回: 200

# go-judge 沙箱
curl http://localhost:5050/version
# 预期返回: {"status":200,"data":{"SandBoxMsg":{"buildVersion":"v1.12.0",...}}}

# hoj-judge 判题服务器
curl http://localhost:8088/version
# 预期返回: {"status":200,"msg":"运行正常","data":{"SandBoxMsg":{...}}}
```

**端到端判题验证**（可选）：

```bash
# 登录 → 提交代码 → 检查结果
python -c "
import urllib.request, json
# 登录
req = urllib.request.Request('http://localhost:6688/api/login',
    data=json.dumps({'username':'root','password':'hoj123456'}).encode(),
    headers={'Content-Type':'application/json'})
resp = urllib.request.urlopen(req, timeout=10)
token = resp.headers.get('Authorization')
# 提交（读取整数并输出，无尾部换行）
code = '#include<stdio.h>\nint main(){\n    int a;\n    scanf(\"%d\",&a);\n    printf(\"%d\",a);\n    return 0;\n}\n'
req2 = urllib.request.Request('http://localhost:6688/api/submit-problem-judge',
    data=json.dumps({'pid':'12','code':code,'language':'C','cid':0,'isRemote':False}).encode(),
    headers={'Content-Type':'application/json','Authorization':token})
r = json.loads(urllib.request.urlopen(req2, timeout=30).read())
print(f'submitId: {r[\"data\"][\"submitId\"]}')
"
# 等待 10 秒后查询结果，status=0 表示 Accepted
```

## 六、已知问题

### 1. content-service 和 section-service 数据库表缺失

**现象**：访问 `/api/posts/` 或 `/api/sections/` 返回 500

**原因**：Django 数据库迁移未执行

**解决**：
```bash
cd Open436-ContentService && python manage.py migrate
cd Open436-SectionService && python manage.py migrate
```

### 2. user-service 未注册到 Consul

**现象**：Consul 服务列表中无 user-service

**原因**：服务启动时 Consul 注册可能失败

**解决**：重启服务或检查 Consul 连接

### 3. HOJ 远程判题账号未配置

**现象**：hoj-backend 日志显示远程判题账号未配置

**影响**：仅影响远程判题（如 Codeforces、HDU），本地判题正常

### 附：HOJ 判题系统技术说明

HOJ 判题链路：

```
用户提交代码
  → hoj-backend (6688) 接收，写入 judge 表，推入 Redis 队列
  → hoj-judge (8088) 消费队列，从 MySQL 读取测试用例，写入 /judge/test_case/
  → go-judge (5050) 编译 + 执行代码
  → hoj-judge 比对输出，更新 judge 表状态
  → hoj-backend 返回结果给前端
```

**关键配置项**（已在 docker-compose.yml 中配置完成）：

| 配置 | 说明 |
|------|------|
| `go-judge: privileged: true` | go-judge 需要特权模式访问 cgroup |
| `hoj-testcase` 共享 volume | hoj-judge 写入测试用例，go-judge 读取 |
| `hoj-judge-app.jar` volume 挂载 | 已做二进制 patch，将 `localhost:5050` 替换为 `judgehost:5050` |
| `hoj-judge: entrypoint` | 动态写入 `/etc/hosts` 将 `judgehost` 解析到 go-judge IP |
| `HOJ_DB_PUBLIC_HOST: hoj-mysql` | 覆盖 JudgeServer 默认的旧 MySQL IP |

**默认账号**：root / hoj123456

**详细修复记录**：参见 `HOJ-Fix-Report.md`

## 七、常用运维命令

### Docker 服务

```bash
# 查看容器状态
docker compose ps

# 查看服务日志
docker compose logs -f [服务名]

# 重启单个服务
docker compose restart [服务名]

# 停止所有 Docker 服务
docker compose --profile hoj down

# 停止并删除数据卷（危险！会清除数据）
```

## 八、网络架构

```
浏览器
  ↓
localhost:3000 (Frontend - 本地 npm)
  ├── /           → Vue SPA
  ├── /api/auth/  → Kong:8000 → auth-service (本地)
  ├── /api/...    → Kong:8000 → 各业务服务 (本地)
  └── /algo/      → hoj-vue:80 → HOJ 前端 (Docker)

localhost:3001 (Admin - 本地 npm)
  ├── /           → Vue SPA
  ├── /api/ai/    → ai-service:8008 (本地)
  └── /api/hoj/   → hoj-backend:6688 (Docker)

Docker 网络 (open436-net):
  ├── postgres, redis, consul, kong, minio
  ├── etcd, milvus-standalone (AI 向量数据库)
  └── hoj-mysql, go-judge, hoj-backend, hoj-judge, hoj-vue
```
