# Open436 项目启动流程报告

## 一、项目架构概览

Open436 教育平台采用微服务架构，通过 Docker Compose 编排所有服务。

### 服务清单

| 分类 | 服务名 | 端口 | 说明 |
|------|--------|------|------|
| **基础设施** | postgres | 55432 | 主数据库 |
| | redis | 6380 | 缓存 |
| | consul | 8500 | 服务注册发现 |
| | kong + kong-db | 8000/8001 | API 网关 |
| | minio | 9000/9001 | 对象存储 |
| **HOJ 系统** | hoj-mysql | 3308 | HOJ 专用数据库 |
| | hoj-backend | 6688 | HOJ 后端 API |
| | hoj-judge | 8088 | 判题服务 |
| | hoj-vue | 8066 | HOJ 前端 |
| **业务服务** | auth-service | 8081 | 认证服务 |
| | enrollment-service | 8084 | 报名服务 |
| | user-service | 8002 | 用户服务 |
| | content-service | 8003 | 内容服务 |
| | comment-service | 8004 | 评论服务 |
| | section-service | 8005 | 板块服务 |
| | file-service | 8007 | 文件服务 (Rust) |
| **前端** | frontend | 3000 | 用户前端 |
| | admin | 3001 | 管理后台 |

## 二、启动流程

### 步骤 1：启动基础设施

```bash
docker compose up -d postgres redis consul kong-db kong minio
```

等待所有服务健康：
```bash
docker compose ps postgres consul kong
```

预期状态：
- postgres: `healthy`
- consul: `healthy`（leader elected）
- kong: `healthy`

### 步骤 2：启动 HOJ 系统

```bash
docker compose up -d hoj-mysql hoj-backend hoj-judge hoj-vue
```

验证：
```bash
# HOJ 后端 API
curl http://localhost:6688/api/get-problem-list
# 预期返回: {"status":200,"data":{"records":[],"total":0,...},"msg":"success"}

# HOJ 前端
curl -o /dev/null -w "%{http_code}" http://localhost:8066/algo/
# 预期返回: 200
```

### 步骤 3：启动业务服务

```bash
docker compose up -d auth-service enrollment-service user-service content-service comment-service section-service file-service
```

验证核心服务：
```bash
# Auth 服务
curl http://localhost:8081/actuator/health
# 预期返回: {"status":"UP",...}

# Enrollment 服务
curl http://localhost:8084/actuator/health
# 预期返回: {"status":"UP",...}
```

### 步骤 4：启动前端

```bash
docker compose up -d frontend admin
```

验证：
```bash
# 用户前端
curl -o /dev/null -w "%{http_code}" http://localhost:3000/
# 预期返回: 200

# 管理后台
curl -o /dev/null -w "%{http_code}" http://localhost:3001/
# 预期返回: 200

# HOJ 通过前端代理
curl -o /dev/null -w "%{http_code}" http://localhost:3000/algo/
# 预期返回: 200
```

## 六、已知问题

### 1. content-service 和 section-service 数据库表缺失

**现象**：访问 `/api/posts/` 或 `/api/sections/` 返回 500

**原因**：Django 数据库迁移未执行，`posts` 和 `sections` 表不存在

**解决**：
```bash
docker exec open436-content python manage.py migrate
docker exec open436-section python manage.py migrate
```

### 2. user-service 未注册到 Consul

**现象**：Consul 服务列表中无 user-service

**原因**：服务启动时 Consul 注册可能失败，但服务本身正常运行

**影响**：通过 Kong 网关的请求可能无法路由到 user-service

**解决**：重启服务
```bash
docker compose restart user-service
```

### 3. file-service 健康检查显示 unhealthy

**现象**：`docker compose ps` 显示 file-service 为 `unhealthy`

**原因**：Docker 健康检查配置使用了错误的端口/路径

**影响**：无，服务本身正常运行，`/health` 端点返回 200

### 4. HOJ 远程判题账号未配置

**现象**：hoj-backend 日志显示 POJ/CF/SPOJ 等远程判题账号未配置

**影响**：仅影响远程判题，本地判题正常

## 七、常用运维命令

```bash
# 查看所有容器状态
docker compose ps

# 查看服务日志
docker compose logs -f [服务名]

# 重启单个服务
docker compose restart [服务名]

# 重建并重启服务
docker compose up -d --build [服务名]

# 停止所有服务
docker compose down

# 停止并删除数据卷（危险！会清除数据）
docker compose down -v
```

## 八、网络架构

```
浏览器
  ↓
localhost:3000 (Frontend nginx)
  ├── /           → Vue SPA
  ├── /api/auth/  → Kong:8000 → auth-service
  ├── /api/...    → Kong:8000 → 各业务服务
  └── /algo/      → hoj-vue:80 → HOJ 前端

localhost:3001 (Admin nginx)
  ├── /           → Vue SPA
  └── /api/hoj/   → hoj-backend:6688 (rewrite)

localhost:8066 (HOJ Vue 直连)
  └── /algo/      → HOJ 前端
```
