# Daily Digest Review

> 更新时间：2026-06-03
> 当前状态：MVP Beta 可试用
> 目标：说明技术日报生成系统的当前质量基线、已完成改进和后续优化方向。

---

## 当前结论

日报生成系统已经达到 MVP Beta 试用标准：可以从管理端触发，自动采集多类信息源，完成去重、质量过滤、AI 整理、结构化保存和公开展示。

当前质量不是最终形态，主要问题集中在信息源质量、重复来源控制和自动优化反馈强度上。系统已经具备质量评分与趋势记录，但“把弱点建议强制反馈到下一轮采集策略”的闭环还需要继续增强。

## 已验证能力

- 手动创建日报任务。
- 工作日定时生成。
- 单页、关键词、日报任务共用 crawler 基础能力。
- 多来源采集：搜索引擎、配置 URL、RSS/URL 类型来源。
- 内容去重：URL 规范化、标题相似度、内容指纹。
- 质量过滤：来源可信度、内容长度、结构质量、广告/营销特征。
- AI 整理：生成结构化章节、摘要、要点、来源引用。
- 质量评估：总分、章节得分、弱点、建议、趋势记录。
- 结果保存：后端保存结构化日报，前端可展示。

## 质量基线

最近一次全量验证：

- Crawler 测试：`python -m pytest -q --tb=short`，1266 passed，1 warning。
- Backend 测试：`mvn test`，75 tests passed。
- Frontend build：`npm run build` 通过。

质量达标判断：

- 日报能生成完整 Markdown 和结构化 JSON。
- 每个核心章节至少有可读条目。
- 来源 URL 可追溯。
- 失败来源不会导致整个任务失败。
- 质量评估能给出可执行弱点建议。

## 当前已完成的优化

- 日报和普通采集任务共用 crawler 的去重、质量评分和搜索 fallback。
- 增加 crawler dependency mode 配置，降低外部依赖变化带来的整体不可用风险。
- 管理端系统配置可以修改 AI/crawler/digest/optimization 相关配置。
- Crawler service 支持独立 HTTP 调用，不只依赖博客系统。
- 自动优化记录可以查询，用于追踪质量趋势。

## 仍需改进的问题

| 问题 | 影响 | 下一步 |
|------|------|--------|
| GitHub Trending 可能产生重复聚合 URL | 开源项目章节信息密度不足 | 展开到 repo 级 URL，并按 repo 去重 |
| `tech_article` 偶尔抓到泛化定义页或营销页 | 技术深度不足 | 增强内容源黑名单和标题/正文质量规则 |
| 弱点建议尚未完全变成强策略约束 | 自动优化闭环偏弱 | 将弱点映射到下一轮关键词、site、时间范围和来源优先级 |
| 长文深度摘要稳定性仍依赖模型 | 质量有波动 | 加强 prompt schema 和 fallback 摘要模板 |
| 质量趋势展示还偏基础 | 管理端判断成本较高 | 增加趋势图、最近失败原因、章节质量对比 |

## 后续评审标准

每次调整日报质量后，需要至少检查：

1. 是否能生成日报任务。
2. 每个章节是否有有效来源。
3. 是否存在明显重复 URL 或重复标题。
4. 是否存在泛化定义页、下载页、营销页。
5. AI 生成内容是否引用了真实来源。
6. 自动优化是否记录本轮弱点和下一轮建议。
7. 前端管理页面是否能查看任务状态、结果和质量信息。
